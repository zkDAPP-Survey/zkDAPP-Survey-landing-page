01 — About the project

Replacing centralized survey agencies with cryptographic truth

Public opinion surveys are a cornerstone of democratic societies, yet centralized agencies suffer from trust and integrity issues. Operators may manipulate samples, suppress results, or fail to prevent Sybil attacks.

zkDAPP-Survey uses zkSNARK zero-knowledge proofs to allow any voter to prove eligibility — citizenship, age, region — without revealing their identity. All results are publicly verifiable on-chain.

The platform uses a gas sponsorship model based on EIP-7702, where survey creators can pre-fund participation for the first N voters. After that sponsored quota is used, the survey remains available and the mobile app still handles all proof generation entirely on-device.

  • 01Eliminate trust in centralized entities by using a public blockchain for transparent vote tallying
  • 02Prevent Sybil attacks through unique user verification based on government-issued verifiable credentials
  • 03Incorporate user attributes — age, geolocation, education — for filtered eligibility and result interpretation
  • 04Maintain full voter privacy using zkSNARKs generated client-side on the voter's smartphone
  • 05Allow survey creators to sponsor gas costs for the first N participants while keeping the survey open beyond the sponsored quota

03 — System architecture

How it all fits together

Five components work in sequence — from credential issuance to on-chain vote tallying — with zero personal data exposed at any step.

01
Credential Issuer
  • Issues VC/PID credentials
  • Reviews submitted documents
  • Verifies applicant information
  • Issues credentials to the wallet
→
02
Identity Wallet
  • Stores verifiable credentials
  • Selective disclosure (SD-JWT)
  • Identity key pair generation
  • Signs vote messages
→
03
zkDAPP-Survey App
  • Survey creator + voter UI
  • Client-side ZK proof generation
  • Circom circuits
  • React Native
→
04
Smart Contracts
  • Creates surveys for the organization
  • Verifies zk-proofs
  • Adds voters to the census
  • Handles sponsored voting payments
→
05
Vocdoni
  • Vochain consensus
  • Nullifier enforcement
  • Vote tallying on-chain
  • Anonymous voting protocol
→

SPRINT 01
Oct – Nov 2024
Research & Technology Exploration
Deep-dive into zkSNARK primitives, Circom circuit design fundamentals, and the EVM ecosystem. Evaluated zkSNARK libraries (snarkjs, Circom vs. ZoKrates). Surveyed related work in decentralized voting and digital identity. Defined the system threat model and privacy requirements.
zkSNARKsResearchThreat Model
SPRINT 02
Nov – Dec 2024
System Design & Architecture
Designed the high-level platform architecture: credential issuer, Identity wallet integration, smart contract layer, and backend relay. Defined the ZK circuit specification for eligibility proofs. Documented the nullifier-based double-vote prevention and EIP-7702 gas sponsorship model.
ArchitectureEIP-7702ZK Circuits
SPRINT 03
Jan – Feb 2025
Smart Contracts & Credential Issuer
Implemented SurvFactory and SurvInstance Solidity contracts with VerifierRegistry and IdentityIssuerRegistry. Built the custom credential issuer with VCK IssuerAgent and OpenID4VCI. Forked the Identity wallet with extended signing capabilities — Poseidon hashing, identity key pair generation.
SolidityIdentity WalletCredential IssuanceOpenID4VCI
SPRINT 04
Feb – Mar 2025
ZK Circuits & Mobile App UI
Wrote and tested Circom eligibility circuits: VC signature verification via Merkle inclusion proof, attribute range checks (age, region, education), and vote-choice signing. Prototyped the React Native mobile application with all major screens and voting flows.
CircomReact NativeMobile UI
SPRINT 05
Mar – Apr 2025
Integration & Backend Relay
Connected all components end-to-end: Identity Wallet ↔ survey app ↔ backend relay ↔ smart contracts. Implemented the off-chain vote package validator and gas pool reimbursement logic. Integrated Vocdoni protocol for census management and nullifier enforcement. Initial performance tests of client-side proof generation on mobile hardware.
